Migrationsstrategi
  1. Skapa interfaces + types baserat på befintlig kod
  2. Wrappa befintlig kod som Open-provider (inga nya API:er ännu)
  3. Byt ut direktanrop i komponenter → registry-anrop
  4. Verifiera att allt funkar exakt som innan
  5. Lägg till logging-wrapper
  6. Implementera Google-adapter
  7. Implementera Mapbox-adapter
  8. A/B + Settings UI

Säkerhetsnät

  • Open-providern wrappas först — appen ska fungera identiskt innan vi lägger till nya providers
  • Google/Mapbox-adapters byggs en i taget — inte allt på en gång
  • API-nycklar hanteras server-side (som idag med Anthropic-nyckeln) via env vars
  • Om en provider-nyckel saknas → automatisk fallback till Open
