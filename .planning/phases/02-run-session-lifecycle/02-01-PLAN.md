    **Part B — Hook implementation (GREEN):**
    Create `src/hooks/useRunSession.ts`:

    1. **State types:**
       ```typescript
       type RunStatus = 'idle' | 'active' | 'paused' | 'completed';
       type RunAction =
         | { type: 'START'; runId: string; startTime: number }
         | { type: 'PAUSE'; elapsedMs: number }
         | { type: 'RESUME' }
         | { type: 'END'; elapsedMs: number; distanceMeters: number }
         | { type: 'RECOVER'; snapshot: ActiveRunSnapshot }
         | { type: 'RESET' };
       ```

    2. **Pure reducer `runReducer`:** Export for testing. Guarded transitions only allow valid state changes. Invalid transitions return state unchanged.

    3. **`computeDistance(trace: FilteredPosition[]): number`:** Export for testing. Haversine sum of consecutive trace segments. Use the same haversine formula from gps-filter.ts (inline copy, same pattern as Phase 1).

    4. **`useRunSession` hook:**
       - Uses `useReducer(runReducer, initialState)` for state
       - `useRef` for mutable data: `traceRef` (FilteredPosition[]), `startTimeRef`, `pausedDurationRef`, `pauseStartRef`, `elapsedRef`
       - `useState` for `elapsedMs` (UI display, updated by timer interval) and `distanceMeters` (updated on GPS accept)

       **Transition functions:**
       - `startRun(routeId: string | null)`: Guard status==='idle'. Generate ID via `crypto.randomUUID()`. Dispatch START. Acquire wake lock + setup visibility re-acquire. Start snapshot schedule. GPS watcher and timer started by useEffect reacting to status.
       - `pauseRun()`: Guard status==='active'. Compute elapsed. Record pauseStart. Dispatch PAUSE. Stop snapshot schedule. Release wake lock.
       - `resumeRun()`: Guard status==='paused'. Add paused duration. Dispatch RESUME. Re-acquire wake lock. Restart snapshot schedule.
       - `endRun()`: Guard status==='active'||'paused'. Compute final elapsed + distance. Dispatch END. Stop snapshot schedule. Release wake lock. Build CompletedRun, persist via `dbPut('runs', completedRun)`. Return the CompletedRun.
       - `recoverRun(snapshot: ActiveRunSnapshot)`: Guard status==='idle'. Dispatch RECOVER. Restore refs from snapshot (startTime, trace, elapsed, pausedDuration). Then call startRun-like side effects (wake lock, snapshots, GPS watcher via status change).
       - `reset()`: Dispatch RESET. Clear all refs.

       **Side effects (useEffect keyed on status):**
       - **GPS effect:** When status==='active', call `watchFilteredPosition`. onAccepted: push to traceRef, update distanceMeters state, call `onPointAccepted(getSnapshot)`. Cleanup: `navigator.geolocation.clearWatch(watchId)`.
       - **Timer effect:** When status==='active', setInterval at 100ms. Each tick: `elapsedRef.current = Date.now() - startTimeRef.current - pausedDurationRef.current`, then `setElapsedMs(elapsedRef.current)`. Cleanup: clearInterval.
       - **Cleanup on unmount:** Stop snapshot schedule, release wake lock, clear GPS watcher.

       **getSnapshot helper (for crash recovery):** Reads from refs to build an ActiveRunSnapshot. Used by snapshot schedule and onPointAccepted.

       **Return type:**
       ```typescript
       interface UseRunSessionReturn {
         status: RunStatus;
         elapsedMs: number;
         distanceMeters: number;
         trace: FilteredPosition[];
         startRun: (routeId: string | null) => void;
         pauseRun: () => void;
         resumeRun: () => void;
         endRun: () => Promise<CompletedRun>;
         recoverRun: (snapshot: ActiveRunSnapshot) => void;
         reset: () => void;
       }
       ```

    **Important implementation notes:**
    - Use `useRef` for trace array to avoid re-renders on every GPS point. Expose `trace` via a getter that returns `traceRef.current`.
    - Timer uses wall clock math (Date.now() - startTime - pausedDuration), NOT interval counter increments.
    - Voice navigation stop/resume on pause is NOT in this hook — that is UI-level (NavigationView handles it in Plan 02).
    - The `runReducer` and `computeDistance` MUST be exported (named exports) so the test file can import them directly.
