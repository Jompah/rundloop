## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Hero metric | 48px (text-5xl) | 700 (bold) | 1.0 | Current rolling pace value |
| Metric value | 32px (text-3xl) | 700 (bold) | 1.1 | Distance, time, remaining distance values |
| Metric label | 12px (text-xs) | 500 (medium) | 1.3 | "PACE", "DISTANCE", "TIME", "REMAINING" labels |
| Unit suffix | 16px (text-base) | 400 (regular) | 1.0 | "/km", "km", inline unit indicators |

Rationale: Hero metric at 48px is readable at arm's length while running (METR-05). 32px secondary metrics are large enough for peripheral glance. 12px labels are small but always UPPERCASE for scanability.

## Component Inventory

### RunMetricsOverlay

The single new UI component for this phase.

**Layout:** 2x2 grid overlay anchored to the bottom half of the map viewport. Replaces the existing bottom bar inline stats (`formatElapsed | distance km`) with a dedicated metrics panel.

```
+---------------------------------------+
|  MAP (visible through overlay gap)    |
|                                       |
+---------------------------------------+  <-- overlay top edge (~50% viewport)
|                                       |
|   CURRENT PACE          AVG PACE      |
|   5:30                  5:45          |
|   /km                   /km avg       |
|                                       |
|   DISTANCE       |     TIME           |
|   3.2 km         |     25:30         |
|                  |                    |
|   REMAINING             (progress)    |
|   1.8 km          ================   |
|                                       |
| [Voice] [Pause] [End]                |
+---------------------------------------+
```

**Grid structure:**
- Row 1: Two columns. Left = hero pace (current rolling). Right = average pace (smaller).
- Row 2: Two columns. Left = distance covered. Right = elapsed time.
- Row 3: Full width. Remaining distance + progress bar.
- Row 4: Existing run controls (Pause/Resume/End) — unchanged from NavigationView.

**Visual treatment:**
- Background: `bg-gray-900/90 backdrop-blur-sm` (matches existing NavigationView panels)
- Border top: `border-t border-gray-800` subtle separator from map
- Safe area: `safe-bottom` class for iOS home indicator padding
- Rounded top corners: `rounded-t-2xl` to distinguish overlay from map

**States:**

| State | Visual Treatment |
|-------|-----------------|
| Active (running) | Full opacity, accent-colored pace, timer counting |
| Paused | `opacity-60` on metrics grid, values frozen, "PAUSED" label visible |
| Insufficient data | Pace shows "--:--", other metrics show "0.0" / "00:00" |
| Route complete (remaining = 0) | Remaining shows "0.0 km", progress bar full |

No destructive actions in this phase's new UI. The End Run button exists but was specified in Phase 2 and triggers the existing EndRunDialog confirmation flow.

## Interaction Contracts

### Metric Update Cadence
- Timer (elapsed time): Updates every 100ms, driven by existing useRunSession setInterval
- Distance: Updates on each GPS position callback (typically every 1-3 seconds)
- Rolling pace: Computed on each render driven by the 100ms timer, reads from trace ref
- Remaining distance: Derived from distance, updates at same cadence

### Transitions
- Metric value changes: No animation. Values snap to new numbers. Running at pace, smooth animations are not perceivable and add rendering cost.
- Pause/resume: `transition-opacity duration-300` for the opacity-60 dimming effect
- Initial mount: No entry animation. Overlay is present from the start of the run.

### Touch Targets
- Run control buttons (Pause/Resume/End): Minimum 44px height, unchanged from Phase 2
- No tappable elements in the metrics grid itself — read-only display
- Voice toggle button: Existing 44px target, unchanged
