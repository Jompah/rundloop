  <action>
    **Create `src/lib/gps-filter.ts`** with these exports:

    1. **Type** `FilterRejectionReason = 'accuracy' | 'jitter' | 'teleport'` — exported for consumers to understand rejections

    2. **Pure function** `shouldAcceptPosition(newPos: GeoPosition, lastAccepted: FilteredPosition | null): { accepted: boolean; reason?: FilterRejectionReason }`:
       - If `newPos.accuracy > 30`: return `{ accepted: false, reason: 'accuracy' }`
       - If `lastAccepted` is null: return `{ accepted: true }` (first point always accepted if accuracy OK)
       - Calculate distance via haversineMeters between `lastAccepted` and `newPos`
       - If distance < 3: return `{ accepted: false, reason: 'jitter' }`
       - Calculate time delta: `(newPos.timestamp - lastAccepted.timestamp) / 1000`
       - If timeDelta > 0 and `distance / timeDelta > 12.5` (45 km/h): return `{ accepted: false, reason: 'teleport' }`
       - Otherwise: return `{ accepted: true }`

    3. **haversineMeters**: Try to import from `src/lib/storage.ts`. If Plan 01 has run and the function is exported, use that import. If Plan 01 hasn't run yet (parallel wave 1), include an inline copy of the haversine formula as a **private** function. The inline copy is the canonical version in this case — Plan 01's export is the canonical version when both plans have completed. Use the same formula from the existing codebase (R=6371000, standard haversine).

    4. **Stateful wrapper** `watchFilteredPosition(onAccepted: (pos: FilteredPosition) => void, onRejected: (pos: GeoPosition, reason: FilterRejectionReason) => void, onError: (err: GeolocationPositionError) => void): number`:
       - Maintains `lastAccepted: FilteredPosition | null = null` as closure state
       - Calls `watchPosition` from `geolocation.ts`
       - For each position: calls `shouldAcceptPosition`
       - If accepted: creates `FilteredPosition` from GeoPosition (strip heading), updates `lastAccepted`, calls `onAccepted`
       - If rejected: calls `onRejected` with position and reason
       - Returns the watchId from underlying `watchPosition` call

    5. Import `GeoPosition` from `./geolocation` and `FilteredPosition` from `@/types`
  </action>

    **Create `src/lib/wake-lock.ts`** with:

    1. **Module-level state**: `let wakeLock: WakeLockSentinel | null = null`

    2. **`isWakeLockSupported(): boolean`**: Returns `typeof window !== 'undefined' && 'wakeLock' in navigator`

    3. **`acquireWakeLock(): Promise<boolean>`**:
       - If `!isWakeLockSupported()`: return false
       - Try `wakeLock = await navigator.wakeLock.request('screen')`
       - Add release listener: `wakeLock.addEventListener('release', () => { wakeLock = null })`
       - Return true on success, false on catch

    4. **`releaseWakeLock(): void`**:
       - Call `wakeLock?.release()`
       - Set `wakeLock = null`

    5. **`setupVisibilityReacquire(): () => void`**:
       - Create handler: `const handler = async () => { if (document.visibilityState === 'visible' && !wakeLock) { await acquireWakeLock(); } }`
       - Add `document.addEventListener('visibilitychange', handler)`
       - Return cleanup function: `() => document.removeEventListener('visibilitychange', handler)`

    **Note on Wake Lock warning banner (CONTEXT.md decision):** The CONTEXT.md locks the decision "If Wake Lock API unavailable (older iOS), show warning banner."
