## Interaction Contracts

### MAP-01: Auto-Center on GPS

| Step | Behavior |
|------|----------|
| App opens, stored position exists (< 24h old) | Map initializes at stored [lng, lat], zoom 13. State: initializing. |
| App opens, no stored position | Map initializes at [0, 0], zoom 2 (world view). "Locating..." overlay shown. State: initializing. |
| GPS lock acquired | flyTo(userPosition, zoom 15, duration 1500ms, essential: true). State transitions to centered. "Locating..." overlay fades out (200ms opacity transition). |
| GPS updates in centered mode | easeTo(userPosition, zoom 15, bearing 0, pitch 0, duration 300ms). Map smoothly follows. |

### MAP-02: Center-on-Me Button

| Step | Behavior |
|------|----------|
| User drags map (in centered mode) | State transitions to free-pan. Center-on-me button fades in (150ms opacity transition). |
| User taps center-on-me button | State transitions to centered. Map flyTo(userPosition, zoom 15, duration 800ms). Button fades out (150ms). |
| User is in navigation mode | Center-on-me button hidden. Existing recenter button handles this flow. |

### MAP-03: Centering State Machine

| Transition | From | To | Trigger | Map Effect |
|-----------|------|-----|---------|------------|
| GPS acquired | initializing | centered | GPS_LOCK | flyTo (1500ms) |
| User drags | centered | free-pan | USER_PAN (dragstart event) | none (user controls map) |
| Tap center button | free-pan | centered | RECENTER | flyTo (800ms) |
| Start run | centered / free-pan | navigating | START_NAVIGATION | yield control to NavigationView |
| Stop run | navigating | centered | STOP_NAVIGATION | flyTo to user position, north-up |
| User drags during nav | navigating | navigating (showRecenter=true) | USER_PAN | none (handled by existing NavigationView) |
| GPS position update | any (except initializing) | same | GPS_UPDATE | easeTo in centered; ignored in free-pan; handled by NavigationView in navigating |
