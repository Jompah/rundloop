   ```
   {view === 'routes' && (

     />
   )}
   ```

   CRITICAL: The onDelete handler must increment historyRefreshKey so RunHistoryView
   re-fetches from IndexedDB and the deleted run disappears from the list:
   ```
   {selectedRun && (
     <RunDetailOverlay
       run={selectedRun}
       onClose={() => setSelectedRun(null)}
       onDelete={(id) => {
         setSelectedRun(null);
         setHistoryRefreshKey((k) => k + 1);
       }}
     />
   )}
   ```

   ```
   const TAB_VIEWS = ['generate', 'history', 'routes', 'map'] as const;
   const showTabBar = (TAB_VIEWS as readonly string[]).includes(view);

   // At the bottom of the JSX, before closing fragment/div:
   {showTabBar && (
     <TabBar
       activeTab={view === 'map' ? 'generate' : (view as 'generate' | 'history' | 'routes')}
       onTabChange={(tab) => setView(tab)}
     />
   )}
   ```
   Note: when view is 'map' (showing a generated route on the map), the "Map" tab should appear active.

7. Remove the old HistoryView import and the old `onClick={() => setView('history')}` button from the generate view (it becomes a tab now). Keep the settings button.

8. Remove the import of the old HistoryView component.

9. Ensure the generate view, settings view, navigate view, summary view blocks are unchanged (only add tab bar and new views).
